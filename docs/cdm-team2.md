# 临床数据管理二室内容

## 团队简介

当前团队共17人，其中1人处于试用阶段，其他16人数据管理工作经验均超过3年。
团队是一支跨临床场景的复合型团队，融合了三类核心背景人才：传统数据管理背景，临床监查员（CRA）背景，临床协调员（CRC）背景。成员自带临床团队的协同经验，能快速建立和监查、CRC 团队的沟通机制，推动数据问题快速闭环。过去几年中开展内部交叉培训，让传统数据管理背景的成员补充临床场景知识，让临床背景的成员提升数据系统能力，让经验复用，打造人人懂临床、人人懂数据的全能型团队，并综合利用3种经验，实现 “前端预防 - 过程管控 - 后端闭环” 的全流程数据质量管控。

## 核心职责

1. 负责临床试验数据质量全生命周期管理，聚焦关键数据与关键流程：包括主要终点/次要终点、入排标准、疗效指标、安全性数据、重要方案偏离、关键流程合规性等；
2. 开展数据质量风险信号识别与分析：数据完整性与一致性分析、逻辑异常识别、质疑管理分析、数据管理报告等，识别数据质量系统性问题，推动根本原因分析（RCA）与整改闭环；
3. 开展跨职能协同与沟通，定期向运营、医学、统计团队汇报数据质量发现，与临床质量保证部门持续对接，对系统性/高风险问题推动中心稽查与源数据质控；
4. QC小组：创建数据管理质控流程并常态化开展工作合规校验，确保各项数据管理动作严格对标 SOP，守住流程合规底线；集中排查共性、批量问题，精准定位日常工作中的薄弱环节，实现问题早发现、早整改；反向输出优化建议，驱动 SOP 迭代完善，形成 “执行 - 质控 - 优化” 的良性闭环。
5. CRF标准库小组：本组严格对标CDISC行业标准，落地 CRF 全维度标准化建设，从源头满足合规要求、保障数据质量，有效降低核查与递交风险。深度践行 QbD 理念，依托标准库实现模板、字段、逻辑、布局统一，消除人为差异，保障各项目 CRF 设计质量稳定一致。为公司顺利完成NMPA、FDA等国内外药监数据递交工作筑牢质量根基。
6. DVS 标准化小组：致力于 DVS 全流程标准化管理，保障部门整体项目数据质量的一致和可控；与技术团队合作开发EC标准库管理平台和DRS线上管理平台，从核查逻辑撰写、审核、发布到落地实现高效流程和质量管理； 核查逻辑标准化将减少重复配置与临时规则梳理工作，大幅提升数据核查、质疑管理的整体工作效率和质量。
